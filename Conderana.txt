Comandos explicados:
* git clone - Se emplea para clonar repositorios en un nuevo directorio. En nuestro caso esta acción
              se llevó a cabo desde Get from VCS, sin emplear el comando.
* git status - Muestra los estados de los archivos del directorio de trabajo, cuáles se han modificado,
               creado o borrado y los archivos que no están siendo considerados para el próximo commit,
               en nuestro caso Conderana.txt se ha modificado y no está siendo considerado para el commit.

               Logs:
                    PS C:\Users\Elena\IdeaProjects\hello-world> git status
                    On branch main
                    Your branch is up to date with 'origin/main'.

                    Changes not staged for commit:
                      (use "git add <file>..." to update what will be committed)
                      (use "git restore <file>..." to discard changes in working directory)
                            modified:   Conderana.txt

                    no changes added to commit (use "git add" and/or "git commit -a")

* git add - Añade un cambio del directorio de trabajo en el entorno de ensayo, indicando de este modo que en la próxima
            confirmación se quieren incluir actualizaciones en un archivo.
